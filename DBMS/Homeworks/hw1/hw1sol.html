Homework 1 Solutions

I. Independent work.

You must solve problems 1-3 individually without any assistance from anyone.

Problem 1 (25 points). You are given an E/R diagram (see file Diagram_for_problem1.pdf in the Homework 1 folder) and the following excerpt from a description of the domain (Teaching staff):

"...Teaching staff are subdivided into instructors and TAs. Every member of the teaching staff has a unique ID and a name. Every instructor holds a position within the department, for instance, as an associate professor, visiting professor, etc. Each member of the teaching staff is assigned to exactly one course. Each course has a unique number and a name."

Find at least five mistakes in the diagram; note that each instance of bad design counts as a mistake. Submit the diagram with your corrections. You must assume that all information (including constraints) on the domain that must be reflected in the diagram, is mentioned in the excerpt above. You get five points for each corrected mistake; if correcting a mistake requires several changes to the diagram, only all changes together will get you five points. Just the corrections are enough for full credit, provided you have corrected the mistakes. (For partial credit, you may submit English explanations of the mistakes.) You will get extra credit if you find mistake(s) your instructor has not anticipated.

Solution:

  1. "Courses" is not a weak entity and should not be marked as such.
  2. Redundancy: remove the "TAs" subclass and add an attribute (for instructor/TA type of staff) to Teaching Staff.
  3. The "Teaching" relation should have a rounded arrow in the direction from "Teaching staff" to “Courses" to express the referential-integrity constraint that everyone who is teaching courses is assigned to exactly one course.
  4. The "ID" attribute of the entity set "Teaching staff" should be underlined to denote that it is a unique key for "Teaching staff".
  5. In the diagram, the "Instructors" subclass should have an attribute for department position.

II. Collaborative work.

Problem 4 (25 points total). Run the sample JDBC program Books.java (in the Homework 1 folder). (Note: If you run the program more than once, it will not be able to create the BOOKS table anew after the first run; you can remove the BOOKS tables between the runs by typing "drop table BOOKS;" in MariaDB.)

After running the program, type the following queries in MariaDB:

select * from BOOKS where price > 9.00;

select * from BOOKS where price <= 9.00;

(4.1) (14 points) Submit the answers to the queries that you get in MariaDB. Explain the meaning of each query (no more than 20 words per query); submit your explanations.

Solution:

SQL> select * from BOOKS where price > 9.00;
 
BOOK_TITLE                               ID      PRICE  AVAILABLE
-------------------------------- ---------- ---------- ----------
All about DBMS                           17      13.49          5
Jack the Ripper                          13       9.99          1
Napoleon and Blucher                      5       9.99          0
 
 
SQL> select * from BOOKS where price <= 9.00;
 
 
BOOK_TITLE                               ID      PRICE  AVAILABLE
-------------------------------- ---------- ---------- ----------
Queen Lucia                              72       5.99          0
A Calendar of Sonnets                   101       3.49         15
 
 

Meaning of the first query: retrieve all tuples, with all attributes included, from BOOKS, where the price is greater than 9.00.

Meaning of the second query: retrieve all tuples, with all attributes included, from BOOKS, where the price is at most 9.00.

(4.2) (11 points) Write a query to obtain the information on all books where the book title is "Queen Lucia". Submit your query and the answer to the query in MariaDB.
